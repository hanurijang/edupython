# 마크다운(Markdown) 작성 규칙 및 스타일 가이드 종합 가이드

마크다운(Markdown)은 텍스트 기반의 경량 마크업 언어로, 복잡한 서식 기호 없이도 구조화된 문서를 쉽고 빠르게 작성할 수 있도록 고안되었습니다. 본 가이드는 표준 마크다운과 GFM(GitHub Flavored Markdown)을 아우르는 핵심 문법, 올바른 작성 규칙, 그리고 문서의 가독성을 높이는 스타일 가이드를 상세히 다룹니다.

---

## 1. 제목 (Headers)

문서의 구조를 잡는 가장 중요한 요소입니다. `#` 기호의 개수로 레벨(1~6)을 결정합니다.

### 1.1 구문 예시
# H1 제목 (문서 제목, 대주제)
## H2 제목 (중주제, 섹션 제목)
### H3 제목 (소주제, 서브 섹션)
#### H4 제목
##### H5 제목
###### H6 제목

### 1.2 권장 규칙 (Style Guide)
* **공백 필수:** `#` 기호와 제목 텍스트 사이에는 반드시 **한 칸의 공백(Space)**을 두어야 합니다. (잘못된 예: `#제목`)
* **줄 바꿈:** 제목 위쪽에는 **한 줄의 빈 줄**을 두어 시각적 가독성을 확보하는 것이 좋습니다.
* **순차적 사용:** 제목 레벨을 건너뛰지 마세요. `## H2` 다음에 바로 `#### H4`로 넘어가는 것은 문서의 논리적 구조를 해칩니다.

## 2. 텍스트 강조 (Emphasis)

텍스트의 특정 부분을 강조하여 독자의 시선을 사로잡을 수 있습니다.

### 2.1 구문 예시
*기울임(Italic)* 또는 _기울임_
**굵게(Bold)** 또는 __굵게__
***굵고 기울임*** 또는 ___굵고 기울임___
~~취소선(Strikethrough)~~ (GFM 지원)

### 2.2 권장 규칙 (Style Guide)
* **일관성 유지:** 기울임이나 굵은 글씨를 사용할 때 `*`와 `_` 중 하나를 선택해 문서 전체에서 일관되게 사용하세요. 보통 `*`와 `**`가 더 널리 사용됩니다.
* **기호 안팎의 공백:** 강조 기호 내부에는 공백이 없어야 합니다. (잘못된 예: `** 텍스트 **`)

## 3. 목록 (Lists)

정보를 열거하거나 단계를 설명할 때 사용하며, 정렬 방식에 따라 두 가지로 나뉩니다.

### 3.1 순서 없는 목록 (Unordered Lists)
* `-`, `*`, `+` 기호를 사용할 수 있습니다. 단, 하나의 문서 안에서는 한 가지 기호로 통일하는 것이 좋습니다. (주로 `-` 권장)

- 사과
- 바나나
  - 바나나우유 (2칸 또는 탭 들여쓰기로 하위 목록 생성)
- 오렌지

### 3.2 순서 있는 목록 (Ordered Lists)
* 숫자와 마침표(`.`)를 사용합니다. 마크다운의 특성상 모든 번호를 `1.`로 적어도 렌더링 시 자동으로 순서가 매겨지지만, 원본 텍스트 가독성을 위해 순서대로 적는 것을 권장합니다.

1. 첫 번째 단계
2. 두 번째 단계
   1. 하위 첫 번째 단계 (들여쓰기 필요)
3. 세 번째 단계

### 3.3 체크리스트 / 작업 목록 (Task Lists)
* GFM에서 지원하는 기능으로, 대괄호 `[]` 안에 공백 또는 `x`를 넣어 사용합니다.

- [x] 완료된 작업
- [ ] 진행 예정 작업

## 4. 링크와 이미지 (Links & Images)

웹 자원이나 로컬 파일을 문서 내에 연결하는 방법입니다.

### 4.1 링크 (Links)
* `[텍스트](URL "툴팁 텍스트")` 형식으로 작성합니다. 툴팁 텍스트는 생략 가능합니다.

[Google 공식 홈페이지](https://www.google.com)
[네이버](https://www.naver.com "NAVER로 이동")

### 4.2 이미지 (Images)
* 링크 구문 앞에 느낌표(`!`)를 붙입니다. 대괄호 안에는 이미지의 **대체 텍스트(Alt Text)**를 반드시 적어주어 웹 접근성을 높여야 합니다.

![마크다운 로고](https://markdown-here.com/img/icon256.png)

### 4.3 참조 링크 (Reference-style Links)
* 문서 내에 URL이 너무 많아 본문이 복잡해질 때 사용합니다.

본문에서 [구글][google-link]과 [네이버][naver-link]를 참조합니다.

[google-link]: https://www.google.com
[naver-link]: https://www.naver.com

## 5. 코드 삽입 (Code)

개발 문서나 기술 블로그 작성 시 필수적인 문법으로, 인라인 코드와 코드 블록으로 구분됩니다.

### 5.1 인라인 코드 (Inline Code)
* 문장 중간에 변수명, 명령어 등을 삽입할 때 백틱(`) 1개로 감쌉니다.

이 프로젝트를 실행하려면 `npm start` 명령어를 입력하세요.

### 5.2 코드 블록 (Code Blocks)
* 여러 줄의 코드를 작성할 때는 위아래를 백틱 3개(```)로 감쌉니다.
* 시작하는 백틱 3개 바로 뒤에 프로그래밍 언어 이름을 소문자로 명시하면 **구문 강조(Syntax Highlighting)**가 적용됩니다.

```javascript
function greet(name) {
    console.log(`Hello, ${name}!`);
}
greet('World');
